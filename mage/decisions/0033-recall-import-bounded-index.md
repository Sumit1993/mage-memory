# 0033 — Recall: `@import` the bounded root index into the host's auto-loaded context (the capture companion to ADR-0032)

> **Status: accepted (ratified 2026-07-01 — impl on `main`, riding 0.0.12).** Recall counterpart to [ADR-0032](0032-capture-redirect-native-memory.md)
> (capture). Output of the 2026-06-25 grill; capture mechanism spike-validated 2026-06-25, recall-load
> validated 2026-06-27 (see ADR-0032 §Gate).

## Context

The ADR-0030 soak measured the *recall* half failing alongside capture: the agent **ignores the
`AGENTS.md` "Read `mage/INDEX.md` first" instruction** and recalls through its own native memory, so
mage's notes go un-surfaced even when a relevant one exists. [ADR-0006](0006-two-layer-recall-per-wing-skills.md)
made recall a two-layer **pull** (root index + per-wing on demand) — correct in shape, but a pull is
volitional, and the soak shows it gets skipped.

The host hands us a deterministic lever (grounded against the memory docs, 2026-06-25): Claude Code's
**`@path` import** — `CLAUDE.md` files import other files, **expanded and loaded into context at
launch**, resolved relative to the importing file, recursive to a **4-hop** max. Two properties make
it the right recall vector:

- It converts a *volitional pull* ("read INDEX") into a *deterministic launch-load*.
- **Project-root `CLAUDE.md` (and its imports) survives `/compact`** — re-read from disk and
  re-injected — so recall persists across the user's continuously-compacted single-chat workflow,
  which the SessionStart digest does not.

The binding constraint is **context cost**: imports *load in full and do not reduce context*. So we
import only what is **bounded**. mage's index is already two files:

- `INDEX.md` — the **bounded root** (a *wings* map: "N notes across 1 wing → `_index.mage.md`",
  measured **266 B** in 2026-07; grows with **wings** — ≈never — not with notes).
- `_index.mage.md` — the **per-note** list (grows linearly with notes).

This finally realizes the shelved "MEMORY.md bridge" (`notes/agentmemory-mining-map.md` — "realized
via INDEX/AGENTS") — but *deterministically*, and via the bounded root, not a copy.

## Decision

1. **One common intent — "the agent holds mage's bounded index at launch" — one per-harness channel.**
   Symmetric to [ADR-0032](0032-capture-redirect-native-memory.md)'s common capture directive: the
   intent is harness-agnostic; only the *channel* is per-adapter.
   - *Claude Code (primary):* with `autoMemoryDirectory` = `resolveDocsRoot().root` (this repo: `mage/`;
     hub models place it at the docs-root top, [ADR-0032 §Hub models](0032-capture-redirect-native-memory.md)),
     mage emits **`<root>/MEMORY.md`** — a **CC-adapter twin** of the portable, canonical `INDEX.md`
     (not a rename: the host-specific name stays out of the portable core, ADR-0009). CC **auto-loads
     it at launch** (recall-load spike PASSED 2026-06-27: a fresh 2nd session answered canaries
     planted only in `MEMORY.md`, zero tool calls), **self-bounded** by CC's 200-line / 25KB cap — cheap
     without mage enforcing a budget. Because CC self-bounds it, `MEMORY.md` can **fold the single-wing
     `_index.mage.md` per-note list** in for richer recall, while `INDEX.md` stays the strict bounded
     wings-map (clause 4). The `PreToolUse` deny keeps it mage-owned (regenerated by `mage index`).
   - *Other harnesses (fallback):* **`@import mage/INDEX.md`** (the bounded-root wings-map) in the
     KB's `CLAUDE.md`/`AGENTS.md` — `CLAUDE.md → AGENTS.md → INDEX.md` is 2 hops (≤ 4), resolves
     relative to the root file, and survives `/compact`. Loads in full, so it must stay the bounded
     root (never the per-note `_index.mage.md`).
   Either way the agent gets mage's index at launch, replacing the ignored "read INDEX first"
   *instruction* with a deterministic *load*. (On CC, keep the `@import` floor too: `MEMORY.md`
   auto-load is proven at session start, but post-`/compact` re-injection is unverified, and the
   `@import` is documented to survive compaction — belt-and-suspenders for the continuously-compacted
   single-chat workflow.)
2. **Never import the per-note `_index.mage.md` or note bodies.** They grow; they stay
   **pull-on-demand** (ADR-0006 progressive disclosure). The imported root orients the agent to pull
   the right per-wing index / note when relevant.
3. **Capability-adaptive** ([ADR-0009](0009-no-runtime-automation-rides-host-hooks.md)): `@import` is
   Claude Code syntax. Harnesses that support it get the deterministic load; harnesses that don't
   keep the prose "read INDEX" instruction as the floor (harmless literal text there).
4. **Bounded by construction.** The imported file stays a *map*, not a list. `mage index` owns
   `INDEX.md`, so it enforces the cap: a KB that grows to many wings keeps the root a wings/category
   summary (counts, not per-note). Importing a per-note or per-wing index is out of scope — that is
   the unbounded cost this ADR exists to avoid.
5. **Recall channel ≠ recall source.** The host's auto-load is the *channel*; the **canonical,
   git-durable `INDEX.md` is the source** — a pointer imported live, not a copy projected in. No
   duplication, no drift, no schema-map — the distinction that separates this from the cut
   harvest/projection ([ADR-0005](0005-one-canonical-memory-others-are-feeders.md)).

## Gate

Low-risk (a small bounded import), so lighter than ADR-0032's:

> **Correction (2026-07-19, [ADR-0039](0039-context-footprint-measure-and-bound.md)):** the
> "~4KB" figure this gate rested on was wrong — and wrong about the *wrong file*. The `@import`
> root (`INDEX.md`) is genuinely small: 266 B measured. But the **auto-memory twin
> (`MEMORY.md`)**, which §1 deliberately left unbounded on the reasoning that "CC self-bounds
> it", measured **19,291 B — 75% of Claude Code's 25,600 B cap**, growing ~264 B per note. The
> host's self-bound is *silent truncation*, not graceful degradation. ADR-0039 supplies the
> measurement and the budget policy this ADR deferred.

- **Behavioral A/B in the soak:** with the import on, does the agent *apply* a relevant mage note
  more often (pull the right note, avoid re-deriving) than with it off? **KILL** if no measurable
  recall lift, or if the bounded import is found to meaningfully bloat launch context.
- Ship-and-watch is acceptable given the bounded cost; the crown is the same live soak reject-ledger.

## Consequences

- Recall stops depending on the agent volunteering to read a file; mage's map is *present* every
  session and after every compaction, at a small flat cost (see the Correction above — the
  auto-memory twin's cost is neither small nor flat, and is now bounded by ADR-0039).
- ADR-0032's relocation "recall side-effect" is struck — capture and recall are cleanly separate.
- A new managed line in `AGENTS.md` (`@mage/INDEX.md`) that the AGENTS.md block / `mage connect`
  owns; onboarding notes the prose "read INDEX" as the non-`@import` fallback.

## Open questions

- The size-cap policy for `INDEX.md` once a KB has many wings (when does the root stop listing every
  wing and summarize?).
- Whether to also offer an opt-in per-wing import for small single-wing KBs where the per-note index
  is itself bounded (probably no — keep one rule).
